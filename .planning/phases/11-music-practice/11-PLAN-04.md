# Plan 04: UI Components — Piano Roll, Tuner, Session Log, MusicWorkspace

## Objective

Create the four React components that form the Music workspace UI. PitchRollPanel renders the scrolling chromatic piano roll using Canvas 2D at 60fps, reading pitch data from useMusicStore.getState() synchronously inside the rAF loop (no prop drilling, no React state in hot path). TunerPanel shows the current note and cents deviation with color-coded glow using standard Zustand hook selectors. SessionLogPanel displays session statistics from Python. MusicWorkspace is the root component that composes them and activates pitch detection.

Depends on Plans 02 (workspace types wired) and 03 (usePitchDetector hook + music.store.ts exist).

Imports: useRef, useEffect from 'react'; useMusicStore from '../../store/music.store'; Panel from '../../panels/Panel'.

Constants (define at module level, not inside component):
  MIDI_MIN = 36  (C2)
  MIDI_MAX = 84  (C6)
  VISIBLE_SEMITONES = MIDI_MAX - MIDI_MIN + 1  (= 49)
  C_NOTES = [36, 48, 60, 72, 84]  (MIDI values of C2–C6 for labels)
  NOTE_NAMES_SHARP = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B']

Helper functions at module level:
  pitchColor(cents: number): string
    if Math.abs(cents) <= 10: return 'rgba(0,255,255,0.9)'   // cyan
    if Math.abs(cents) <= 25: return 'rgba(255,176,0,0.9)'   // amber
    return 'rgba(255,0,128,0.9)'                               // magenta

  midiToRow(midi: number, canvasHeight: number): number
    // Y position for a given MIDI note. MIDI_MAX at top (y=0), MIDI_MIN at bottom.
    const rowH = canvasHeight / VISIBLE_SEMITONES
    return (MIDI_MAX - midi) * rowH

  nameToMidi(name: string): number
    // Converts a note name string (e.g. "A4") to its MIDI number (e.g. 69).
    // Define as a module-level const function — NOT exported, internal to the component file.
    const NOTE_NAMES = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B']
    octave = parseInt(name.slice(-1))
    noteName = name.slice(0, -1)
    return NOTE_NAMES.indexOf(noteName) + (octave + 1) * 12
    // Examples: nameToMidi("C2") === 36, nameToMidi("A4") === 69, nameToMidi("C6") === 84

Component PitchRollPanel():
  const canvasRef = useRef<HTMLCanvasElement>(null)
  const rafRef = useRef<number>(0)

  Helper drawGrid(canvas: HTMLCanvasElement, offscreen: HTMLCanvasElement):
    // Draw the static chromatic grid onto the offscreen canvas.
    // Reads CSS tokens via getComputedStyle(document.documentElement) for color vars.
    // Background: var(--bg-elevated) for white keys, var(--bg-base) for black keys.
    // Black key MIDI rows: C#, D#, F#, G#, A# (midi % 12 in [1,3,6,8,10])
    // Semitone grid lines: 1px border-color lines between rows
    // C-note labels: right-aligned text "C2", "C3" etc at each C row
    // Octave separator: slightly brighter line at each C note
    // Grid is drawn once; redrawn only on resize.

  Helper drawFrame(canvas: HTMLCanvasElement):
    const ctx = canvas.getContext('2d')!
    const { width, height } = canvas
    const dpr = window.devicePixelRatio || 1
    const cssW = width / dpr
    const cssH = height / dpr

    // 1. Draw static grid from offscreen
    ctx.drawImage(offscreenRef.current!, 0, 0)

    // 2. Read current pitch from store synchronously — safe in rAF, no React subscription
    const s = useMusicStore.getState()
    const currentNote = s.currentNote   // e.g. "A4" or ""
    const currentCents = s.currentCents // e.g. -12

    // 3. Shift existing pitch trail left by 1px using two-offscreen approach:
    //    Maintain a separate offscreen "trail" canvas. Each frame:
    //    a. Shift trail canvas content left by 1px: trailCtx.drawImage(trailCanvas, -1, 0)
    //    b. Clear rightmost column on trail canvas
    //    c. Paint new rightmost column with latest pitch (if detected)
    //    d. Composite: ctx.drawImage(gridOffscreen, 0, 0) then ctx.drawImage(trailCanvas, 0, 0)

    // 4. Paint rightmost column with latest pitch sample (if any)
    //    Use nameToMidi(currentNote) to convert the note name string to a MIDI number.
    //    Only paint if currentNote is non-empty and MIDI is in [MIDI_MIN, MIDI_MAX]
    if (currentNote) {
      // Convert note name to MIDI using nameToMidi helper defined at module level
      // Then apply pitchColor(currentCents) and ctx.shadowColor/shadowBlur glow
      const color = pitchColor(currentCents)
      // ... derive midi from currentNote via nameToMidi, then:
      // const rowH = cssH / VISIBLE_SEMITONES
      // const y = midiToRow(midi, cssH)
      // ctx.shadowColor = color; ctx.shadowBlur = 6
      // ctx.fillStyle = color; ctx.fillRect(cssW - 2, y, 2, rowH)
      // ctx.shadowBlur = 0
    }

  rAF loop useEffect (empty deps []):
    1. Create offscreen grid canvas (offscreenRef.current = document.createElement('canvas'))
    2. Create trail offscreen canvas (trailRef.current = document.createElement('canvas'))
    3. Set up ResizeObserver on canvasRef to update canvas dimensions + DPR scaling + redraw grid:
       const dpr = window.devicePixelRatio || 1
       canvas.width = entry.contentRect.width * dpr
       canvas.height = entry.contentRect.height * dpr
       canvas.style.width = entry.contentRect.width + 'px'
       canvas.style.height = entry.contentRect.height + 'px'
       ctx.scale(dpr, dpr)
       drawGrid(canvas, offscreenRef.current)
    4. MutationObserver on document.documentElement for data-theme attribute changes:
       on change: redraw grid (reads fresh CSS tokens)
    5. Start rAF: function frame() { drawFrame(canvasRef.current!); rafRef.current = requestAnimationFrame(frame) }
       rafRef.current = requestAnimationFrame(frame)
    6. Cleanup: cancelAnimationFrame(rafRef.current); resizeObs.disconnect(); mutObs.disconnect()

Render:
  return (
    <Panel id="pitch-roll" title="Pitch Roll" noPad>
      <canvas ref={canvasRef} style={{ width: '100%', height: '100%', display: 'block' }} />
    </Panel>
  )
