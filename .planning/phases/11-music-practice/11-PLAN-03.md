# Plan 03: Audio Engine — AudioWorklet + pitchfinder Hook

## Objective

Build the audio detection layer: the AudioWorklet processor (plain JS in ui/public/), the React hook that manages AudioContext lifecycle and runs pitchfinder on the main thread, and the music Zustand store that holds session state. This plan produces the reusable primitives that Plan 04 (UI panels) will consume.

Depends on Plan 02 (WorkspaceId type must exist so music.store.ts can import it without errors). Does not depend on Plan 01 (Python backend is independent).

Step 2 — Create d:\Zuki\ui\public\pitch-processor.js (plain JavaScript, not TypeScript):

This file runs in AudioWorkletGlobalScope. It CANNOT use ES module imports — pitchfinder cannot be imported here. The correct pattern is to send raw Float32Array blocks back to the main thread via port.postMessage, where pitchfinder runs normally.

File content structure:
- Class PitchProcessor extends AudioWorkletProcessor
- constructor() calls super()
- process(inputs, outputs, parameters) method:
  - Read channel = inputs[0]?.[0]
  - If channel has length, call this.port.postMessage(channel.slice()) to send a copy of the 128-sample Float32Array to main thread
  - Return true (keep processor alive)
- registerProcessor('pitch-processor', PitchProcessor) at module bottom

The file must NOT contain any import or require statements. It must NOT attempt to use pitchfinder. It only captures audio samples and forwards them.

Place at: d:\Zuki\ui\public\pitch-processor.js — Vite serves ui/public/ as static files at the root path, so the browser loads it via /pitch-processor.js when addModule('/pitch-processor.js') is called.

FILE 1 — Create d:\Zuki\ui\src\store\music.store.ts:

Zustand store (import { create } from 'zustand' — same pattern as workspace.store.ts).

PitchEvent interface (export):
  type: "pitch_event"
  note: string        (e.g. "A4")
  midi: number        (0–127, e.g. 69 for A4)
  frequency: number   (Hz)
  cents: number       (offset from nearest semitone, negative=flat, positive=sharp)
  confidence: number  (0.0–1.0; set to 1.0 when pitchfinder gives no confidence value)
  ts: number          (Date.now())

MusicStore interface:
  mode: 'voice' | 'instrument'
  isListening: boolean
  currentNote: string     (e.g. "A4", empty string when silent)
  currentCents: number    (cents deviation, 0 when silent)
  setMode: (mode: 'voice' | 'instrument') => void
  setListening: (v: boolean) => void
  setPitch: (note: string, cents: number) => void

create<MusicStore>: initial values mode='voice', isListening=false, currentNote='', currentCents=0. Implement setMode, setListening, setPitch as set() calls.

Export: export const useMusicStore = create<MusicStore>(...)

Also export the helper functions that usePitchDetector.ts needs (avoids re-implementing):
  export function freqToMidi(freq: number): number
    return Math.round(12 * Math.log2(freq / 440) + 69)

  export function midiToName(midi: number): string
    const NOTE_NAMES = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B']
    return NOTE_NAMES[midi % 12] + Math.floor(midi / 12 - 1)

  export function freqToCents(freq: number, midi: number): number
    const refFreq = 440 * Math.pow(2, (midi - 69) / 12)
    return 1200 * Math.log2(freq / refFreq)

FILE 2 — Create d:\Zuki\ui\src\hooks\usePitchDetector.ts:

Imports: useEffect, useRef from 'react'; Pitchfinder from 'pitchfinder'; bridge from '../bridge/ws'; useMusicStore, PitchEvent, freqToMidi, midiToName, freqToCents from '../store/music.store'.

Export signature: export function usePitchDetector(): void

Internal refs (all useRef, never useState — hot path must not trigger re-renders):
  streamRef: useRef<MediaStream | null>(null)
  ctxRef: useRef<AudioContext | null>(null)
  nodeRef: useRef<AudioWorkletNode | null>(null)
  bufRef: useRef<Float32Array[]>([])
  lastSendRef: useRef<number>(0)

The detector function (pitchfinder): create once inside the effect:
  const detectPitch = Pitchfinder.YIN({ sampleRate: 48000 })

useEffect with empty deps []:
  1. let active = true (strict-mode guard)
  2. async function start():
     a. const stream = await navigator.mediaDevices.getUserMedia({ audio: true, video: false })
     b. If !active: stream.getTracks().forEach(t => t.stop()); return
     c. const ctx = new AudioContext()
     d. await ctx.audioWorklet.addModule('/pitch-processor.js')
     e. const src = ctx.createMediaStreamSource(stream)
     f. const node = new AudioWorkletNode(ctx, 'pitch-processor')
     g. node.port.onmessage = (e: MessageEvent<Float32Array>) => {
          bufRef.current.push(e.data)
          if (bufRef.current.length >= 8) {
            // Merge 8 x 128 = 1024 samples
            const merged = new Float32Array(1024)
            let offset = 0
            for (const chunk of bufRef.current) { merged.set(chunk, offset); offset += chunk.length }
            bufRef.current = []
            const freq = detectPitch(merged)
            if (freq && freq > 60 && freq < 2000) {
              const midi = freqToMidi(freq)
              const note = midiToName(midi)
              const cents = Math.round(freqToCents(freq, midi))
              useMusicStore.getState().setPitch(note, cents)
              const now = Date.now()
              if (now - lastSendRef.current > 200) {
                lastSendRef.current = now
                const payload: PitchEvent = { type: 'pitch_event', note, midi, frequency: freq, cents, confidence: 1.0, ts: now }
                bridge.send('pitch_event', payload)
              }
            }
          }
        }
     h. src.connect(node)
        // Do NOT connect node to ctx.destination — analysis only, no audio playback
     i. streamRef.current = stream; ctxRef.current = ctx; nodeRef.current = node
     j. useMusicStore.getState().setListening(true)
  3. start().catch((err) => console.error('[usePitchDetector]', err))
  4. Return cleanup function:
     active = false
     nodeRef.current?.disconnect()
     streamRef.current?.getTracks().forEach(t => t.stop())
     ctxRef.current?.close()
     useMusicStore.getState().setListening(false)

Frequency gate (freq > 60 && freq < 2000): filters subsonic noise and supersonic false positives. Covers C2 (65.4 Hz) to C7 (2093 Hz) — more than enough for the C2–C6 piano roll range.
